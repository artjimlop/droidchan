droidchan
=========

De forma muy sintética puede decirse que el proyecto consiste en el desarrollo de un tablón de imágenes con aspectos sociales para plataformas móviles con Android. El objetivo inicial era el siguiente: una aplicación en la que cualquier usuario pudiese crear un hilo subiendo una imagen, responder al mismo y valorarlo. Para diferenciarlo de lo ya existente se decidió incluir aspectos como las suscripciones entre usuarios y el envío de mensajes entre los mismos. La idea de crear grupos de usuarios surgió para poner la aplicación al servicio de la comunidad, de modo que no se obliga a recoger temas en categorías ya existentes como en los foros, sino que los usuarios crean ellos mismos grupos en los que comparten contenido de diversa índole. El control de la comunidad no lo ejerce un administrador, si no que se delega en los usuarios.
Para el desarrollo de este proyecto ha sido necesaria la realización de dos sub proyectos: por una parte se ha tenido que crear un servicio web REST para dar soporte a la otra parte, que es la aplicación Android con la que interactuará el usuario. En todo momento se ha procurado seguir el esquema aprendido en la asignatura Ingeniería de Software de Gestión III correspondiente a la titulación de Ingeniería Técnica en Informática de Gestión, ya que en esta asignatura se realizó un servicio web con características similares al presente desarrollo.
El servicio web REST contiene los conectores que gestionan las peticiones al sistema gestor de base de datos que provienen de la aplicación Android y responden con datos en formato JSON. Estos datos son recogidos por los objetos tipo Business de la aplicación, siéndoles mostrados a los usuarios a través de la lógica implementada para controlar cada una de las actividades (pantallas) que conforman la aplicación.
Entrando en detalles más técnicos, para el servicio web ha sido fundamental la librería Jersey (para la creación del servicio REST), mientras que para la aplicación Android ha sido necesaria la librería FTP4j (para el envío y recepción de archivos vía FTP), Android-support-appcompat (para mejorar la experiencia de usuario) y Android Annotations (para reducir la cantidad de código necesaria para recoger las imágenes). En ambos casos ha sido vital el uso de la librería Jackson (para convertir de una clase Java a JSON y viceversa) y Jettison (para leer y escribir archivos JSON).
Tras la realización del proyecto se concluyeron ciertos puntos de evolución los cuales no fueron abordados debido a que ya se había excedido el límite de horas destinadas a la asignatura.
Desde el inicio se hicieron pruebas de integración de los componentes del proyecto paralelamente a la implementación de código. Una vez fueron alcanzados todos los objetivos, se realizaron pruebas de usuario que fueron superadas satisfactoriamente.
El mayor esfuerzo fue realizado al inicio del proyecto ya que fue necesario investigar sobre APIs REST, conceptos de diseño en Android y tratamiento de archivos JSON. La planificación inicial fue cumplida aunque con un exceso de horas tal como se ha comentado anteriormente. La tarea que más retraso ha sufrido fue la implementación de código, ya que se intentaba llevar pareja a la investigación. Debido a la motivación personal por querer aprender a diseñar aplicaciones Android se decidió continuar y cubrir todos los objetivos propuestos en lugar de limitar las funcionalidades y cumplir con la estimación temporal.
